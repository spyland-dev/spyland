# Политика безопасности

## Сообщение об уязвимости

Если вы обнаружили уязвимость безопасности, проблему конфиденциальности или любое поведение,
которое может привести к компрометации пользовательских данных, сообщите об этом приватно.

Пожалуйста, **не создавайте публичные Issues** для уязвимостей,
которые могут подвергнуть пользователей риску.

Предпочтительные способы связи:

* GitHub Private Vulnerability Reporting
* Личное обращение ко мне через GitHub или через мою почту

При сообщении желательно указать:

* Описание проблемы
* Шаги для воспроизведения
* Затронутые версии
* Логи, скриншоты или proof-of-concept (по возможности)
* Предполагаемое влияние уязвимости

Все сообщения рассматриваются добросовестно.

## Поддерживаемые версии

Поскольку проект находится в активной разработке и ещё не достиг стабильного релиза 1.0, исправления
безопасности гарантируются только для актуальной версии разработки.

Более старые версии могут не получать обновления безопасности.

## Модель безопасности проекта
Модель безопасности `spyland` основана на следующих принципах:

- ### Всё локально
Данные активности предназначены для хранения на устройстве пользователя.

- ### Отсутствие телеметрии
`spyland` не предназначен для передачи данных пользовательской активности внешним сервисам.

- ### Открытый исходный код
Весь исходный код доступен для изучения и аудита.

- ### Явные компоненты
Сбор данных осуществляется отдельными бэкендами, которые пользователь устанавливает самостоятельно.

- ### Контроль пользователя
Пользователь самостоятельно определяет, какие бэкэнды установлены и запущены в системе.

## Модель угроз

`spyland` не предназначен для защиты от:
* Полностью скомпрометированной ОС
* Вредоносного ПО, работающего с теми же привилегиями, что и пользователь
* Физического доступа к уже разблокированной пользовательской сессии
* Утечки БД не вызванная проектом или его официальным компонентом
* Вредоносных сторонних клиентов, утилит, или бэкэндов

## Благодарности
Сообщения об уязвимостях, аудиты безопасности, ревью кода и ответственное раскрытие информации
помогают сделать проект лучше и безопаснее для всех пользователей.
